Rapport du projet de compilation


Choix techniques :

On a utilisé le schema de compilation décrit dans sujet 2012.

Les conventions d'appel sont :
 - on passe l'argument dans t0 et la cloture dans t1
 - on renvoie un pointeur dans v0
 - la fonction force prend un pointeur dans v0 et renvoie un pointeur dans v0 (le meme)

On a créé un fichier error.ml qui regroupe toutes les erreurs renvoyées par le compilateur.
Le fichier print.ml permet d'afficher l'arbre du programme lors des différentes étapes de compilation.
Le fichier primitives.ml contient le code MIPS des quatre primitives.


Difficultés rencontrées :

Le déboggage...
La récursivité mutuelle des déclarations globales et locales.
L'évaluation paresseuse, notamment obtenir les bons résultats pour les fichiers tests "lazy".
